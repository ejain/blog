ToonTalk

January 31, 2006 · Humor · Programming

It looks like our initiative to port all of our code to a more compact and efficient language didn’t get very far. So what next?

Perhaps we should try a simpler and more “friendly” programming language: ToonTalk.

Variables are called boxes, function calls are flying birds and strings are text pads.

Instead of a stack there is Dusty the Vacuum, instead of comparator functions you use scales, and object creation is done by construction trucks.

Best of all: Instead of automatic garbage collection (which everone knows is decadent), you need to blow up stuff to free memory.
